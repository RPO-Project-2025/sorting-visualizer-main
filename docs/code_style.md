# Стандарты кода

## Именование переменных
- snake_case для переменных и функций
- PascalCase для классов
- UPPER_CASE для констант

## Структура проекта
- Каждая команда работает в своей папке
- Общие утилиты в папке `common/`
- Документация в `docs/`

## Коммиты
- Используйте понятные сообщения
- Формат: `[тип] описание`
- Типы: feat, fix, docs, style, refactor, test
